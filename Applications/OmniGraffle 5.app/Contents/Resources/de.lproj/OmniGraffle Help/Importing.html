Importieren aus anderen Programmen

OmniGraffle kann mehrere verschiedene Dateiformate verarbeiten. Sie können diese Dateien importieren, indem Sie sie wie eine gewöhnliche OmniGraffle-Datei öffnen.
Microsoft® Visio®-Dokumente, -Vorlagen und -Schablonen (.vsd, .vdx, .vst, .vtx, .vss, .vsx)
Dokumente, die mit der Microsoft® Visio®-Diagrammsoftware in deren XML-basiertem Dateiformat gesichert wurden (und jetzt auch solche im Original-Binärformat), werden von OmniGraffle Pro gelesen. Sie können das Binärformat zwar öffnen, Sie können jedoch nicht in dieses exportieren. Sie müssen die Dateien daher als OmniGraffle-Diagramm speichern oder in das XML-Format exportieren. Beachten Sie weiterhin, dass Grafiken im Microsoft® Enhanced Metafile (EMF)-Format nicht importiert werden können.
DOT-Dateien (.dot)
OmniGraffle verarbeitet eine Untergruppe der von Graphviz verwendeten DOT-Sprache. Obwohl es sich nicht um eine vollständige Implementierung von DOT handelt, kann diese Funktion sehr nützlich sein.
Diagram! 2 Dokumente (.diagram2 und .dpalette2)
Diagram ist ein NeXTSTEP/OPENSTEP-Programm ähnlich OmniGraffle. OmniGraffle kann die Objekte einer Diagram-Datei importieren. Dies gilt jedoch nicht für zugehörige(s) EPS-Bilder, Audiomaterial oder Links.
OmniOutliner-Dokumente (.ooutline und .oo3)
Beim Öffnen einer OmniOutliner-Datei werden die Objekte der Gliederung als Diagramm interpretiert. Ein Dialogfenster wird angezeigt. Geben Sie darin an, wie die Gliederung interpretiert werden soll:
Verwenden Sie das Einblendmenü um einen Diagrammstil auszuwählen.
Weisen Sie Objekten im Diagrammstil mithilfe der Tabelle Spalten in der Gliederung zu. Wenn Sie mehrere Spalten mit Informationen zu jedem Objekt anzeigen möchten, verwenden Sie einen Diagrammstil mit gruppierten Objekten. Sie können auch den Notizen des Objekts eine Spalte zuweisen.
Geben Sie an, ob Sie die Gliederung in ein neues Dokument oder in eine neue Arbeitsfläche in dem geöffneten Dokument einfügen möchten.
Xcode-Projekte (.xcode), Project Builder Projekte (.pbproj) und Frameworks (.framework)
OmniGraffle verwendet die Objective-C Kopfzeilen, um ein Diagramm der Beziehungen zwischen Vor- und Nachfahren, Protokollen und Kategorien zu erstellen.
Interface Builder nib-Dateien (.nib)
OmniGraffle stellt die Verknüpfungen in der nib-Datei als Diagramm dar und sorgt so für eine einfache Visualisierung.
EOModeler-Dateien (.eomodeld)
OmniGraffle erstellt ein Diagramm der in der Datei beschriebenen Datenbankbeziehungen.
Ordnerstrukturen
Bewegen Sie einen Ordner aus dem Finder auf das OmniGraffle Programmsymbol, um den Inhalt des Ordners als Diagramm darzustellen.
