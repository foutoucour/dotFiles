Importation à partir d’autres applications

Différents formats de fichier sont pris en charge par OmniGraffle. Vous pouvez importer ces fichiers en les ouvrant comme si vous ouvriez un fichier OmniGraffle normal.
Documents, modèles et patrons Microsoft® Visio® (.vsd, .vdx, .vst, .vtx, .vss, .vsx)
Les documents enregistrés par le logiciel de création de schéma Microsoft® Visio® dans son format de fichier XML (et désormais aussi dans son format de fichier binaire d’origine) sont lisibles dans OmniGraffle Pro. Si vous pouvez ouvrir le format binaire, vous ne pouvez pas exporter vers ce format ; vous devrez l’enregistrer sous un schéma OmniGraffle ou l’exporter au format XML. Les graphiques au format Microsoft® Enhanced Metafile (EMF) ne peuvent pas être importés.
Fichiers DOT (.dot)
OmniGraffle comprend un sous-ensemble du langage DOT utilisé par Graphviz. Il ne s’agit pas d’une implémentation complète du DOT, mais elle peut néanmoins s’avérer suffisamment exploitable.
Les documents Diagram! 2 sont également reconnus (.diagram2 et .dpalette2)
Diagram est une application NeXTSTEP/OPENSTEP similaire à OmniGraffle. OmniGraffle peut importer les objets dans un fichier Diagram, mais ne peut pas importer les images EPS qui y sont jointes, les sons et les liens.
Documents OmniOutliner (.ooutline et .oo3)
Lorsque vous ouvrez un fichier OmniOutliner, les éléments de la structure sont interprétés en tant que schéma. Une zone de dialogue apparaît à l’écran ; c’est alors que vous devez y indiquez comment interpréter exactement la structure :
Utilisez la liste locale pour choisir un style de schéma.
Utilisez le tableau pour attribuer des objets du style de schéma aux colonnes de la structure. Si vous souhaitez afficher plusieurs colonnes d’informations pour chaque élément, utilisez un style de schéma qui possède des objets groupés. Vous pouvez également attribuer une colonne aux notes des objets.
Vous pouvez placer la structure dans un document totalement nouveau ou dans un nouveau canevas du document ouvert.
Projets Xcode (.xcode), projets Project Builder (.pbproj) et frameworks (.framework)
OmniGraffle utilise les en-têtes Objective C pour élaborer un schéma des relations d’hérédité, de protocoles et de catégories.
Fichiers nib Interface Builder (.nib)
OmniGraffle trace un graphique des connexions du fichier nib pour faciliter la visualisation.
Fichiers EOModeler (.eomodeld)
OmniGraffle crée un schéma des relations de la base de données décrites par le fichier.
Structures des dossiers
Faites glisser un dossier à partir du Finder et déposez-le sur l’icône de l’application OmniGraffle afin de faire ressortir les données du dossier sous forme de schéma.
